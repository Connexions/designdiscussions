Archive Read-Only API

The Archive Read-Only API is for retrieving content with the expected result being that the content is displayed to a user. The API should allow the user to retrieve HTML content. Resources are retrieved by the browser.

API URIs

  • Retrieve Content: archive.cnx.org/contents/[UUID]
    • Version number can be specified: archive.cnx.org/contents/[UUID[@version]]
    • No version results in latest version being returned.  It will be redirected to the /[UUID]@[Version] url.
  • Retrieve Resource: archive.cnx.org/resources/[resource-UUID]/[type]
  • Export Files: archive.cnx.org/exports/[UUID[@version]]/[type]
    • no version redirects to the versioned URL for that export


The [UUID] is the unique identifier of the content. The Version is expected to be a hash. The [type] is the type of export file such as pdf, epub, complete (complete zip), offline (offline zip) or cnxml. Resource URLs are added to the content at publish time and should be retrieved by the browser and not a separate API call.

Web View Responsibilities

  • content title as part of URL (SEO/human friendly)
    • be careful about user-provided content going into the URL
      • UTF-8
      • Min/Max character limit
      • non-URL friendly characters (punctuation)
  • rename export files before delivering them to the user
    • Content-Disposition header
