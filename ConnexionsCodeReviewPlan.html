Connexions Code Review Plan

  1. Programmers will be assigned a component to review (Editor, Templates, Archive API, etc.)
  2. Review code when a pull request is made to move code from a branch into Master of a repository.
  3. What to check (the goal is maintainable code)
    1. Check that it makes sense and does not reinvent a wheel we already have.
    2. Follow the Rhaptos Coding Standards
    3. Check that a LICENSE file exists.
    4. Are variable names understandable?
    5. Are function names descriptive?
    6. Are script/class names descriptive?
    7. Functions should do one thing or dispatch to other functions that do one thing. (Scripts should not be one long function. Â Makes testing more difficult.)
    8. Do templates have more than display logic?
    9. Comments?
  4. What not to check
    1. Do not check if the code was written as you would have written it.
  5. Be nice.
  6. If you get a notice from a code review, correct it as soon as you can.
  7. Once you have corrected the items in a code review, commit them and update the pull request. Â The pull request will then be merged after review.
