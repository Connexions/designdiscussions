Transformation Services Simplified Requirements

NOTE: This document covers the first iteration of the Export file part of Transformation Services.  We are short circuiting this development so we can get CCAP PDF generation off our current Production System.

General


Transformation Services should create these export files

  • PDF
  • EPUB
  • CNXML
  • HTML Zip
  • Complete Zip

Each of the exports should...
  • reuse as much of the existing Connexions code as possible
  • run as a separate service

Messaging Queue


All requests should be received by a messaging queue.  Requests are picked up off of the queue and dispatched to the correct Handler for processing.  The queue should store messages so queue failure does not result in lost requests.  

Export API


The Export API should be a separate process from the rest of the services.  It should...

  • accept a request to generate any of the files
  • return a success message when the package is received.  
  • allow generation of a PDF for previewing
  • deal with dependencies for file generation

A request should include
  • What to generate
  • Where the data is
